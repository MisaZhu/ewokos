## 1. 系统概述

EwokOS 是一个微内核架构的嵌入式操作系统，支持 ARM、AArch64、RISC-V 和 x86_64 多种处理器架构。本文档以 QEMU virt 平台为例，详细讲解系统从启动到正常运行的全过程。

### 1.1 项目结构

```
/workspace
├── kernel/                    # 内核核心代码
│   ├── kernel/src/           # 内核源码（进程、调度、IPC等）
│   ├── kernel/include/       # 内核头文件
│   ├── dev/                  # 设备驱动抽象层
│   ├── lib/                  # 内核库函数
│   ├── loadinit/             # init进程加载器
│   └── platform/             # 架构相关配置
├── machine.virt/             # virt平台BSP
│   └── kernel/bsp/           # 平台特定实现（UART、Timer、IRQ等）
├── system/                   # 用户空间系统
│   ├── basic/               # 基础系统程序
│   │   ├── bin/             # 基础命令（shell、ls等）
│   │   ├── sys/             # 系统服务（init、core、vfsd）
│   │   ├── drivers/         # 用户态驱动
│   │   └── libc/            # C库实现
│   ├── gui/                 # 图形界面组件
│   ├── network/             # 网络组件
│   └── xwin/                # 窗口系统
└── README.md                 # 项目说明
```

### 1.2 核心设计理念

- **微内核架构**：内核只提供最基本的服务（进程调度、IPC、内存管理），其他功能以用户态服务进程形式实现
- **IPC驱动**：所有系统服务通过IPC（进程间通信）机制交互
- **多核支持**：支持SMP多核处理器，具备完善的核间同步机制
- **模块化设计**：驱动、文件系统、网络协议栈等均在用户态实现

## 2. virt平台启动流程

### 2.1 启动阶段概览

virt平台的启动流程分为以下几个关键阶段：

```
硬件复位 → Boot阶段 → 内核初始化 → 加载init进程 → 系统服务启动 → 用户交互
```

### 2.2 Boot阶段（汇编入口）

启动代码位于 [start.c](file:///workspace/machine.virt/kernel/bsp/start.c)，主要完成：

#### 2.2.1 页表初始化

```c
void _boot_start(void) {
    boot_pgt_init();
    // 映射内核代码段：虚拟地址0x40000000 → 物理地址0x40000000
    set_boot_pgt(0x40000000,  0x40000000, 64*MB, 0);
    // 映射内核高端地址：KERNEL_BASE → 物理地址0x40000000
    set_boot_pgt(KERNEL_BASE, 0x40000000, 64*MB, 0);
    // 映射MMIO设备空间：MMIO_BASE → 物理地址0x8000000
    set_boot_pgt(MMIO_BASE,   0x8000000,  256*MB, 1);
    load_boot_pgt(startup_page_dir);
}
```

**ARM架构页表配置**：
- 使用Section类型的一级页表（每项映射1MB）
- 内核代码映射为可读写执行
- MMIO设备空间标记为设备类型（非缓存）

**AArch64架构页表配置**：
- 使用两级页表（L1目录 + L2块描述符）
- 每个块描述符映射2MB
- 支持设备内存属性配置

#### 2.2.2 链接脚本分析

链接脚本 [mkos.lds.S](file:///workspace/machine.virt/kernel/mkos.lds.S) 定义内存布局：

```
base_address = 0x80000000;  // 内核加载基地址

内存布局：
├── .init section      (启动代码，4KB对齐)
├── SVC栈空间          (4KB)
├── .text section      (代码段)
├── .data section      (数据段)
├── .bss section       (未初始化数据)
├── IRQ栈              (4KB)
├── ABT栈              (4KB)
├── UNDEF栈            (4KB)
├── SVC栈              (4KB)
└── 内核栈顶            (_kernel_sp)
```

### 2.3 内核C入口初始化

内核主初始化函数 `_kernel_entry_c()` 位于 [kernel.c](file:///workspace/kernel/kernel/src/kernel.c)：

#### 2.3.1 初始化流程详解

```c
void _kernel_entry_c(void) {
    // 1. 清除BSS段
    memset(_bss_start, 0, (size_t)(_bss_end - _bss_start));
    
    // 2. 系统信息初始化
    sys_info_init();
    
    // 3. 复制中断向量表到物理地址0x0
    copy_interrupt_table();
    
    // 4. 初始化内核虚拟内存映射
    init_kernel_vm();
    
    // 5. UART初始化（早期调试输出）
    uart_dev_init(19200);
    
    // 6. 内核动态内存分配器初始化
    kmalloc_init();
    
    // 7. SD卡驱动初始化
    sd_init();
    
    // 8. 加载内核配置文件
    load_kernel_config();
    
    // 9. 重新映射内核内存空间
    reset_kernel_vm();
    
    // 10. 初始化可分配物理内存
    init_allocable_mem();
    
    // 11. 内核事件队列初始化
    kev_init();
    
    // 12. DMA、信号量、IRQ、共享内存初始化
    dma_init();
    semaphore_init();
    irq_init();
    shm_init();
    
    // 13. 进程管理子系统初始化
    procs_init();
    
    // 14. 加载init进程
    load_init_proc();
    
    // 15. 创建idle进程（每核一个）
    kfork_core_halt(0);
    
    // 16. SMP多核启动（如果启用）
    #ifdef KERNEL_SMP
    for(uint32_t i=1; i<_sys_info.cores; i++) {
        kfork_core_halt(i);
        start_core(i);
    }
    #endif
    
    // 17. 设置定时器中断频率
    timer_set_interval(0, _kernel_config.timer_freq);
    
    // 18. 启用中断，进入调度循环
    __irq_enable();
    halt();  // 主核进入idle状态
}
```

#### 2.3.2 内核虚拟内存映射

`init_kernel_vm()` 函数建立完整的内核地址空间：

```c
static void init_kernel_vm(void) {
    // 1. 初始化4K内存块分配器
    kalloc_reset();
    kalloc_append(KERNEL_PAGE_DIR_BASE+PAGE_DIR_SIZE, KERNEL_PAGE_DIR_END);
    
    // 2. 设置内核页表映射
    set_kernel_vm(_kernel_info.kernel_vm);
    
    // 3. 切换到新的页表基址
    set_translation_table_base(V2P(_kernel_info.kernel_vm));
}
```

**内核地址空间布局**：

| 区域 | 虚拟地址范围 | 说明 |
|------|-------------|------|
| 中断向量 | 0xFFFF0000 | 映射到物理地址0x0 |
| 内核代码 | KERNEL_BASE | 内核镜像 |
| 页表目录 | KERNEL_PAGE_DIR_BASE | 进程页表存储 |
| vsyscall信息 | KERNEL_VSYSCALL_INFO_BASE | 用户态快速系统调用信息 |
| kmalloc区域 | KMALLOC_BASE | 内核动态分配 |
| MMIO空间 | _sys_info.mmio.v_base | 设备寄存器映射 |

### 2.4 加载init进程

init进程加载由 [loadinit.c](file:///workspace/kernel/loadinit/sd/loadinit.c) 实现：

```c
int32_t load_init_proc(void) {
    const char* prog = "/sbin/init";
    int32_t sz;
    
    // 1. 从SD卡ext2文件系统读取init ELF文件
    char* elf = sd_read_ext2(prog, &sz);
    
    // 2. 创建第一个进程
    proc_t *proc = proc_create(TASK_TYPE_PROC, NULL);
    strcpy(proc->info.cmd, prog);
    proc->info.uid = -1;  // 特权进程
    
    // 3. 切换到新进程的地址空间
    set_translation_table_base(V2P(proc->space->vm));
    
    // 4. 加载ELF文件到进程内存
    int32_t res = proc_load_elf(proc, elf, sz);
    
    return res;
}
```

### 3.2 调度器

调度器代码位于 [schedule.c](file:///workspace/kernel/kernel/src/schedule.c)。

#### 3.2.1 调度算法

EwokOS 使用简单的优先级队列调度：

```c
int32_t schedule(context_t* ctx) {
    uint32_t core = get_core_id();
    proc_t* idle_proc = _cpu_cores[core].idle_proc;
    
    // 1. 从就绪队列获取下一个进程
    proc_t* next = proc_get_next_ready();
    
    // 2. 如果没有就绪进程，运行idle进程
    if(next == NULL && idle_proc != NULL) {
        next = idle_proc;
    }
    
    // 3. 切换到目标进程
    if(next != NULL) {
        next->info.state = RUNNING;
        proc_switch(ctx, next, false);
        return 0;
    }
    
    // 4. 无可运行进程，系统panic
    printf("Panic: none proc to be scheduled!\n");
    halt();
    return 0;
}
```

#### 3.2.2 进程切换机制

```c
void proc_switch(context_t* ctx, proc_t* to, bool quick) {
    proc_t* cproc = get_current_proc();
    
    // 1. 保存当前进程上下文
    if(cproc != NULL && cproc->info.state != UNUSED) {
        memcpy(&cproc->ctx, ctx, sizeof(context_t));
    }
    
    // 2. 切换地址空间（如果目标进程不同）
    if(cproc != to) {
        set_translation_table_base(V2P(to->space->vm));
    }
    
    // 3. 处理特殊情况（中断、信号、IPC）
    if(to->space->interrupt.state == INTR_STATE_START) {
        // 设置中断处理入口
        to->ctx.pc = to->space->interrupt.entry;
    }
    else if(to->space->signal.do_switch) {
        // 设置信号处理入口
        to->ctx.pc = to->space->signal.entry;
    }
    else if(to->space->ipc_server.do_switch) {
        // 设置IPC服务入口
        to->ctx.pc = to->space->ipc_server.entry;
    }
    
    // 4. 设置目标进程为运行态
    to->info.state = RUNNING;
    set_current_proc(to);
    
    // 5. 恢复目标进程上下文
    memcpy(ctx, &to->ctx, sizeof(context_t));
}
```

3. 检查IPC超时
4. 检查中断超时
5. 触发调度

## 4. 进程管理与调度

### 4.1 进程创建

#### 4.1.1 创建新进程

```c
proc_t *proc_create(int32_t type, proc_t* parent) {
    // 1. 在任务表中找到空闲槽位
    for (uint32_t i = 0; i < _kernel_config.max_task_num; i++) {
        if (_task_table[i] == NULL) {
            _task_table[i] = (proc_t*)kmalloc(sizeof(proc_t));
            index = i;
            break;
        }
    }
    
    // 2. 初始化进程结构
    memset(proc, 0, sizeof(proc_t));
    proc->info.uuid = ++_proc_uuid;
    proc->info.pid = index;
    proc->info.type = type;
    proc->info.state = CREATED;
    
    // 3. 初始化地址空间（进程类型）
    if(type == TASK_TYPE_PROC) {
        proc_init_space(proc);  // 分配独立页表
    }
    
    // 4. 初始化用户栈
    proc_init_user_stack(proc);
    
    return proc;
}
```

#### 4.1.2 fork实现

```c
proc_t* kfork(context_t* ctx, int32_t type) {
    proc_t* cproc = get_current_proc();
    proc_t* child = kfork_raw(ctx, type, cproc);
    
    // 分配到合适的CPU核心
    core_attach(child);
    
    // 如果core服务已就绪，通知core进程
    if(_core_proc_ready && child->info.type == TASK_TYPE_PROC) {
        kev_push(KEV_PROC_CREATED, cproc->info.pid, child->info.pid, 0);
    }
    
    return child;
}
```

**Copy-on-Write机制**：

```c
static int32_t proc_clone(proc_t* child, proc_t* parent) {
    // 将父进程的页面映射到子进程（只读）
    for(uint32_t p=0; p<pages; ++p) {
        uint32_t phy_page_addr = resolve_phy_address(parent->space->vm, v_addr);
        map_page_ref(child->space->vm, v_addr, phy_page_addr, AP_RW_R, PTE_ATTR_WRBACK);
        // 同时将父进程页面设为只读
        map_page(parent->space->vm, v_addr, phy_page_addr, AP_RW_R, PTE_ATTR_WRBACK);
    }
    // 写入时触发页错误，复制页面
}
```

### 4.3 进程退出与回收

```c
void proc_exit(context_t* ctx, proc_t *proc, int32_t res) {
    // 1. 设置为僵尸状态
    proc_terminate(ctx, proc);
    
    // 2. 释放子进程
    for (i = 0; i < _kernel_config.max_task_num; i++) {
        proc_t *p = _task_table[i];
        if(p->info.father_pid == proc->info.pid) {
            proc_exit(ctx, p, 0);  // 终止所有子进程
        }
    }
    
    // 3. 唤醒等待的父进程
    proc_wakeup_waiting(proc->info.pid);
    
    // 4. 触发调度
    schedule(ctx);
}
```

## 6. IPC进程间通信

IPC是EwokOS微内核的核心通信机制，代码位于 [ipc.c](file:///workspace/kernel/kernel/src/ipc.c)。

### 6.1 IPC架构

```
┌──────────────┐                    ┌──────────────┐
```

### 6.3 IPC调用流程

#### 6.3.1 客户端发起调用

```c
static void sys_ipc_call(context_t* ctx, int32_t serv_pid, 
                          int32_t call_id, proto_t* arg) {
    proc_t* client_proc = get_current_proc();
    proc_t* serv_proc = proc_get(serv_pid);
    
    // 1. 检查服务端状态
    if(serv_proc->space->ipc_server.entry == 0) {
        ctx->gpr[0] = IPC_ERROR_NO_READY;
        return;
    }
    
    // 2. 创建IPC任务
    ipc_task_t* ipc = proc_ipc_req(serv_proc, client_proc, call_id, arg);
    
    // 3. 设置客户端等待状态
    client_proc->ipc_res.state = IPC_BUSY;
    
    // 4. 触发服务端执行
    proc_ipc_do_task(ctx, serv_proc, client_proc->info.core);
}
```

#### 6.3.2 服务端处理

服务端被唤醒后，执行IPC处理函数：

```c
// 服务端IPC处理循环（用户态）
void ipc_serv_run(ipc_handle_func handle, void* p, 
                  void* loop_data, uint32_t flags) {
    while(1) {
        // 等待IPC请求
        proto_t in, out;
        int pid, cmd;
        
        // 获取调用信息
        ipc_get_call(&pid, &cmd, &in);
        
        // 处理请求
        handle(pid, cmd, &in, &out, p);
        
        // 返回结果
        ipc_set_return(pid, &out);
        
        // 结束本次IPC
        ipc_end();
    }
}
```

#### 6.3.3 客户端获取结果

```c
static void sys_ipc_get_return(context_t* ctx, int32_t pid, 
                                uint32_t uid, proto_t* data) {
    proc_t* client_proc = get_current_proc();
    
    // 如果结果未返回，阻塞等待
    if(client_proc->ipc_res.state != IPC_RETURN) {
        proc_block(ctx, client_proc);
        return;
    }
    
    // 复制返回数据
    if(data != NULL) {
        proto_copy(data, client_proc->ipc_res.data.data, 
                   client_proc->ipc_res.data.size);
    }
    
    // 清除IPC状态
    client_proc->ipc_res.state = IPC_IDLE;
}
```

### 6.4 IPC数据结构

```c

    uint32_t  state;         // IPC_BUSY/IPC_RETURN/IPC_IDLE
    proto_t   data;          // 返回数据
} ipc_res_t;

typedef struct {
    uint32_t  uid;           // IPC唯一标识
    uint32_t  counter;       // 超时计数器

    int32_t   client_pid;    // 客户端PID
```

```c
inline void svc_handler(int32_t code, ewokos_addr_t arg0, 
                         ewokos_addr_t arg1, ewokos_addr_t arg2, 
                         context_t* ctx) {
    __irq_disable();          // 禁用中断
    svc_account(code);        // 统计系统调用
    
    // 快速路径（不需要锁）
    if(svc_is_query_fastpath(code)) {
        _svc_handler(code, arg0, arg1, arg2, ctx);
        return;
    }
    
    // 需要内核锁的操作
    kernel_lock();
    _svc_handler(code, arg0, arg1, arg2, ctx);
    kernel_unlock();
}
```

### 8.2 中断处理流程

```c
// 中断入口（汇编）
// 1. 保存上下文
// 2. 调用C处理函数
// 3. 恢复上下文

int32_t interrupt_send(context_t* ctx, uint32_t interrupt) {
    interrupt_item_t* item = get_interrupt(interrupt);
    proc_t* proc = proc_get_by_uuid(item->handler.uuid);
    
    // 1. 保存当前进程状态
    proc_save_state(proc, &proc->space->interrupt.saved_state, ...);
    
    // 2. 设置中断处理入口
    proc->space->interrupt.interrupt = interrupt;
    proc->space->interrupt.entry = item->handler.entry;
    proc->space->interrupt.state = INTR_STATE_START;
    
    // 3. 切换到目标进程执行中断处理
    proc_switch_multi_core(ctx, proc, cproc->info.core);
}

void interrupt_end(context_t* ctx) {
    proc_t* cproc = get_current_proc();
    
    // 1. 恢复之前保存的状态
    proc_restore_state(ctx, cproc, &cproc->space->interrupt.saved_state, ...);
    
    // 2. 如果有pending中断，继续处理
    if(pending != 0) {
        // 重新设置中断处理
    }
    
    // 3. 重新启用中断
    irq_enable_arch(interrupt);
    
    // 4. 触发调度
    schedule(ctx);
}
```

### 9.1 init进程

```c
int main(int argc, char** argv) {
    // 1. 检查是否为内核加载的特权进程
    if(((int16_t)getuid()) >= 0) {
        return -1;
    }
    
    // 2. 如果不是PID 0，则为核idle进程
    if(getpid() != 0)
        idle();
    
    klog("\n[init process started]\n");
    
    // 3. 启动核心系统服务（从SD卡加载）
    run_before_vfs("/sbin/core");    // 核心服务管理

    // 4. 执行初始化脚本
    switch_root();  // 执行 /etc/init.rd
    
    // 5. 进入idle循环
    while(true) {
        usleep(100000);
    }
}
```

## 总结

EwokOS是一个设计精巧的微内核操作系统，通过IPC机制实现了模块化的系统服务架构。virt平台的启动流程清晰地展示了从硬件复位到完整系统运行的全过程：

1. **Boot阶段**：建立初始页表，切换到内核虚拟地址空间
2. **内核初始化**：初始化各子系统（内存、进程、IPC等）
3. **init进程**：启动系统服务链
4. **系统服务**：通过IPC协作提供完整操作系统功能

这种架构使得内核保持精简，同时具备良好的扩展性和可维护性。
